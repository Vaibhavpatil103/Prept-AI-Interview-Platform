## 🔭 Overview

**Prept** is a full-stack SaaS platform that connects job-seeking engineers (**Interviewees**) with experienced professionals (**Interviewers**) for 1:1 mock interview sessions. It solves the problem of ineffective interview preparation by providing:

- **Live mock interviews** with senior engineers from top companies (Google, Meta, Amazon, etc.)
- **AI-powered feedback reports** generated by Google Gemini after each session
- **Real-time HD video calls** with screen sharing, recording, and transcription
- **A credit-based monetization system** where interviewers earn and withdraw real money

### Who is it for?

| Role | Description |
|------|-------------|
| **Interviewees** | Engineers preparing for technical interviews at top-tier companies |
| **Interviewers** | Experienced professionals who want to earn by sharing their expertise |

## ✨ Features

### 🎯 For Interviewees

- 🔍 **Browse & Filter Interviewers** — Search by category (Frontend, Backend, System Design, DSA, DevOps, Mobile, Behavioral, Full Stack)
- 📅 **Slot-based Booking** — Pick from open availability slots and confirm with one click
- 📹 **HD Video Interviews** — 45-minute sessions with screen sharing, recording, and instant playback
- 💬 **Persistent Chat** — Message your interviewer before and after the call for prep notes and follow-ups
- 📊 **AI Feedback Reports** — Post-interview analysis covering technical depth, communication, problem-solving, strengths, and improvements
- 💳 **Credit-based Plans** — Free (1 credit/mo), Starter ($29 — 5 credits/mo), Pro ($69 — 15 credits/mo) with credit rollover

### 🧑‍💼 For Interviewers

- 🗓️ **Availability Management** — Set custom availability windows from the dashboard
- 🤖 **AI Question Generator** — Get role-specific interview questions generated live by Gemini (DSA, System Design, Behavioral, etc.)
- 💰 **Earnings & Withdrawals** — Earn credits per session, track balance, and submit withdrawal requests with admin email notifications
- 📋 **Appointment Dashboard** — View all upcoming and past sessions with interviewee details and feedback history

### 🔒 Platform-wide

- 🛡️ **Arcjet Security** — Bot protection, rate limiting, and abuse prevention on all critical routes
- 🔐 **Clerk Authentication** — Secure sign-in/sign-up with role-based onboarding flow
- 🌙 **Dark Mode** — Full dark theme with system preference detection
- 📧 **Transactional Emails** — Withdrawal request notifications via Resend + React Email

## 🚀 Getting Started

### Prerequisites

- **Node.js** ≥ 18.x
- **npm** ≥ 9.x (or **pnpm** / **yarn**)
- **PostgreSQL** database (or a [Supabase](https://supabase.com/) project)
- Accounts on: [Clerk](https://clerk.com/), [Stream](https://getstream.io/), [Arcjet](https://arcjet.com/), [Resend](https://resend.com/), [Google AI Studio](https://aistudio.google.com/)

### 1. Clone the repository

```bash
git clone https://github.com/your-username/prept.git
cd prept
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

Create a `.env` file in the project root (see [Environment Variables](#-environment-variables) below):

```bash
cp .env.example .env
```

### 4. Set up the database

```bash
# Generate the Prisma client
npx prisma generate

# Run migrations
npx prisma migrate dev

# (Optional) Seed sample data
npx prisma db seed
```

### 5. Run in development mode

```bash
npm run dev
```

The app will be available at **`http://localhost:3000`**.

### 6. Build for production

```bash
npm run build
npm start
```

### Key Design Decisions

1. **Server Actions over API routes** — All mutations and queries use Next.js Server Actions (`"use server"`) instead of traditional REST endpoints, reducing boilerplate and enabling direct function calls from client components.

2. **Credit-based monetization** — Instead of direct payments per session, a credit system decouples pricing from individual sessions. Interviewees subscribe for monthly credits; interviewers accumulate credits and withdraw earnings.

3. **Clerk for auth + subscriptions** — Clerk handles both authentication and subscription plan management (Free/Starter/Pro) via `has({ plan })` checks, eliminating the need for a separate billing provider.

4. **Arcjet per-user rate limiting** — Rate limits are fingerprinted by Clerk user ID (not IP), preventing abuse while allowing legitimate use from shared networks.

5. **Prisma with connection pooling** — Uses `@prisma/adapter-pg` with PgBouncer-compatible pooled connections for serverless environments, plus a `DIRECT_URL` for migrations.

### Auth Flow

```
User visits /sign-in or /sign-up
        │
        ▼
Clerk handles authentication
        │
        ▼
checkUser() syncs Clerk ↔ DB
  • Creates user if new
  • Allocates monthly credits based on plan
  • Rolls over unused credits
        │
        ▼
Role check → UNASSIGNED?
  ├── Yes → Redirect to /onboarding (pick Interviewee or Interviewer)
  └── No  → Redirect to /dashboard or /explore
```

## 📡 API Reference

Prept uses **Next.js Server Actions** instead of traditional REST endpoints. Below are the key actions organized by domain:

### Booking & Sessions

| Action | File | Description | Auth Required |
|--------|------|-------------|:---:|
| `getInterviewers()` | `actions/explore.js` | List all available interviewers with profiles | ❌ |
| `getInterviewerProfile(id)` | `actions/booking.js` | Fetch detailed interviewer profile + availability | ❌ |
| `bookSlot({ interviewerId, startTime, endTime })` | `actions/booking.js` | Book a session, deduct credits, create Stream call | ✅ |
| `getCallData(callId)` | `actions/call.js` | Get call details + generate Stream user token | ✅ |

### Dashboard & Management

| Action | File | Description | Auth Required |
|--------|------|-------------|:---:|
| `setAvailability({ startTime, endTime })` | `actions/dashboard.js` | Set/update interviewer availability | ✅ |
| `getAvailability()` | `actions/dashboard.js` | Fetch current availability window | ✅ |
| `getInterviewerAppointments()` | `actions/dashboard.js` | List all interviewer sessions | ✅ |
| `getInterviewerStats()` | `actions/dashboard.js` | Credit balance, total earned, sessions completed | ✅ |

### AI & Feedback

| Action | File | Description | Auth Required |
|--------|------|-------------|:---:|
| `generateInterviewQuestions({ category })` | `actions/aiQuestions.jsx` | Generate 6 role-specific questions via Gemini | ✅ |

### Payments

| Action | File | Description | Auth Required |
|--------|------|-------------|:---:|
| `requestWithdrawal({ credits, paymentMethod, paymentDetail })` | `actions/dashboard.js` | Submit withdrawal request (rate limited: 3/hr) | ✅ |
| `getWithdrawalHistory()` | `actions/dashboard.js` | List past withdrawal requests | ✅ |
| `approvePayout({ payoutId, adminPassword })` | `actions/payout.js` | Admin: approve a pending payout | ✅ (Admin) |

### Webhook

| Method | Route | Description | Auth |
|--------|-------|-------------|------|
| `POST` | `/api/webhooks/stream` | Stream event handler (call ended, recording ready) | Stream signature |

## 🚢 Deployment

### Deploy to Vercel (Recommended)

1. **Push to GitHub**:
   ```bash
   git add .
   git commit -m "Initial commit"
   git push origin main
   ```

2. **Import to Vercel**:
   - Go to [vercel.com/new](https://vercel.com/new)
   - Import your GitHub repository
   - Vercel auto-detects the Next.js framework

3. **Set environment variables**:
   - Add all variables from the [Environment Variables](#-environment-variables) section in the Vercel project settings

4. **Configure build settings**:
   - Build Command: `npm run build` (uses `postinstall` to generate Prisma client)
   - Output Directory: `.next` (auto-detected)

5. **Set up database**:
   - Ensure your Supabase project allows connections from Vercel's IPs
   - Run migrations against the production database:
     ```bash
     npx prisma migrate deploy
     ```

6. **Configure webhooks**:
   - Set the Stream webhook URL to `https://your-domain.vercel.app/api/webhooks/stream`
   - Set the Clerk webhook URL if using Clerk webhooks

### Deploy with Docker

```dockerfile
FROM node:18-alpine AS base

WORKDIR /app
COPY package*.json ./
RUN npm ci

COPY . .
RUN npx prisma generate
RUN npm run build

EXPOSE 3000
CMD ["npm", "start"]
```

```bash
docker build -t prept .
docker run -p 3000:3000 --env-file .env prept
```

```bash
git clone https://github.com/your-username/prept.git

```

```bash
git checkout -b feature/your-feature-name
```

- `fix:` — bug fix

```bash
git push origin feature/your-feature-name
```

Open a PR against `main` with a clear description of what changed and why.

- Use [GitHub Issues](https://github.com/your-username/prept/issues)
